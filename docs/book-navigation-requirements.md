# 要件定義: 本風ページ送りナビゲーション

[redesign-plan-trpg-rulebook.md](./redesign-plan-trpg-rulebook.md) と [implementation.md](./implementation.md) の方針のうち、「セクション間の移動」の部分を刷新するドキュメント。両ドキュメントのカラー/紙レイヤー/章立て表記などの方針はそのまま有効。

## 背景

フェーズ4までの実装で「本風レイアウト」を、蒼黒背景の上に浮かぶ紙カード + scroll-snapによるセクション送り、として実装していた。しかしユーザーの意図は「画面いっぱいの本」+「スクロールではなくページ送りで章を移動する」というものであり、解釈がずれていた。実装を一度止め、要件をこのドキュメントに固めてから再設計する。

## 1. 現状スナップショット(2026-07-13時点の実装)

以下は今回の要件変更の対象、または前提となる既存実装の事実確認。

- **カラートークン**: `src/styles/global.css`の`@theme`で`--color-bg`(蒼黒)/`--color-text`(銀白)/`--color-accent`(月光色)/`--color-paper-bg`・`--color-paper-text`(羊皮紙)を定義済み。`.paper`ユーティリティ(`paper-bg.jpg` + `background-blend-mode: multiply`)も定義済み。**変更なし。**
- **現行レイアウト(置き換え対象)**: `src/layouts/Layout.astro`の`<html class="snap-y snap-mandatory">` + 各セクション(`Hero.astro`/`About.astro`/`Social.astro`/`Portfolio.tsx`内の各spread)が`min-h-screen snap-start`。縦スクロールでセクション間を移動する現行モデルは本要件で全廃する。
- **`BookCard.astro`**: 見開きカードコンポーネント。`max-w-5xl`の中央寄せカードに、中央スパイン(影のグラデーション)+ 左右2カラム(`md:grid-cols-2`、`md`未満は1カラムに崩れる)。About/Socialで`<Fragment slot="left">`/`<Fragment slot="right">`として使用中。**フルスクリーン化が本要件の対象。**
- **`TocRibbon.astro`**: 左端固定のしおりタブ。`<input type="checkbox">` + `<label>` + CSS(`group-hover`/`peer-checked`)のみで開閉、JSファイルなし。リンクは`href="#hero"`等のアンカー(スクロールジャンプ前提)。**リンク先の動作を本要件で変更する必要あり。**
- **`Header.astro`**: 左上固定、タイトル+章ナビ。リンクは同じく`href="#about"`等のアンカー。**同上。**
- **`PageTurnHint.astro`**: 右下固定の折り目風ボタン。現状は`window.scrollBy({top: window.innerHeight})`するだけの「次のセクションへスクロール」ボタン。**本要件のページ送りボタンの土台として流用する。**
- **`LoadingScreen.astro`**: 表紙が開くローディング演出。`document.fonts.ready`を待って左右パネルが開く。**変更なし。**
- **`Portfolio.tsx`**: React island(`client:visible`)。`chunkPairs`で作品を2件ずつに分割し、ペアごとに`min-h-screen snap-start`の`<section>`を生成。クリックで中央モーダル表示。**ページ分割ロジック(2件ずつ)自体は流用し、本要件の統一ページ列に組み込む。scroll-snapのみ廃止。**
- **既知の未対応課題(フェーズ5対象、本要件では触らない)**: `Hero.astro`/`Social.astro`/`Footer.astro`に残る旧カラークラス(`text-primary`/`bg-primary`/`text-letter`等、現テーマでは未定義)。`background.jpg`の残骸。

## 2. 新要件

### ナビゲーション方式

- スクロール/`scroll-snap`を**全廃**する。
- ページ送りは「クリック(右下角ボタン=次のページ、左下角ボタン=前のページ)」と「矢印キー」のみ。
- **やらない**: スワイプジェスチャー対応、3D回転(フリップ)アニメーション。

### 遷移演出

- 横スライド。現在ページが左または右にスライドアウトし、次ページが逆側からスライドインする(進む/戻るで方向を変える)。

### BookCardのサイズ

- 現状の`max-w-5xl`カードから、画面いっぱい(フルスクリーン相当)に拡大する。

### 章内自動ページ分割

- Aboutなど、本文量が増えて1画面(1見開き)に収まらない場合、自動的に複数ページに分割する。
- 手動でのテキストチャンク分けは不要。コンテンツ量に応じて自動的にページ数が決まる。
- ページ送り操作で章内の複数ページも順に進む(章をまたぐ操作と地続き)。

### 対象範囲

- Hero(表紙)・About・Portfolio・Socialの**全セクションを1本の連続したページ列**として統一する。
- Portfolioの既存の「作品2件ずつのペア」は、そのまま統一ページ列の中の1ページ(1見開き)として扱う(再設計不要)。

### TOC/Headerナビの挙動

- TOC(しおり)・Headerの章リンクをクリックすると、間のページを送ることなく、目的の章の先頭ページへ**直接ジャンプ**する。

### モバイル(`md`ブレークポイント未満)の挙動

- PCでの1見開き(左ページ+右ページ)は、モバイルでは左ページ・右ページをそれぞれ独立した1ページとして扱う。
- つまり「PCの1見開き = モバイルの2ページ」となる。

### 意図的に含めないもの(スコープ外)

- フリップ(3D回転)アニメーション
- スワイプジェスチャー対応
- Contactセクションの実装(未実装のまま、別タスク)
- フェーズ5の旧資産掃除(旧カラークラス、`background.jpg`)

## 3. 未決事項(次回の実装計画時に検討する)

- **章内自動ページ分割の技術的手法**: CSS多段組(`columns`)による自動フローを使うか、JSでDOMを計測して分割するか、他の方法か。次回の実装設計フェーズで検討する。
- **ページ送りの状態管理の置き場所**: 現行はAstro静的コンポーネント(Header/TocRibbon/About/Social)+ React island(Portfolio)が混在している。統一ページ列の状態(現在ページ・総ページ数・章ごとのページ数)をどこが持つか(単一のReact/vanilla JSコントローラーを新設するか等)は次回検討する。
- **ページ位置インジケーターの要否**: 「1/4ページ」のような現在位置表示を出すかどうかは未確認。要ユーザー確認。
